## 詳細問題分析與修正建議

### 1. 字元編碼顯示問題 (嚴重)
**問題描述**: 檔案中存在多個問號(?)字符，應為中文字符但編碼錯誤

**影響記錄範例**:
- 行號40: `桃園市新屋區,?榔里,8鄰9號,...,?榔社區活動中心,?榔里`
- 行號51: `彰化縣員林市,三義里,...,?淑娟,8347171`
- 行號66: `彰化縣彰化市,茄苳里,...,寶?里、茄南里`
- 行號75: `彰化縣彰化市,阿夷里,...,下?里、興北里`

**修正建議**:
```bash
# 使用iconv轉換編碼
iconv -f big5 -t utf-8 避難收容處所點位檔案v9.csv > 避難收容處所點位檔案v9_fixed.csv
```
或手動修正特定字符：
- `?榔里` → `永榔里`
- `?淑娟` → `林淑娟` 
- `寶?里` → `寶獅里`
- `下?里` → `下埔里`

### 2. 欄位缺失問題 (中等)
**問題描述**: 部分記錄的村里欄位為空

**影響記錄範例**:
- 行號2: `1,新竹縣,,,121.073,24.386,...` (村里欄位缺失)
- 行號29: `28,臺東縣臺東市,,營區位置恕不敘明,...` (村里欄位缺失)
- 行號96: `95,屏東縣滿州鄉,,福德路64巷3號,...` (村里欄位缺失)

**修正建議**:
- 根據地址資訊補填村里名稱
- 如無法確定，標記為"未知"或留空但需記錄

### 3. 適用災害類別欄位缺失 (中等)
**問題描述**: 部分記錄的適用災害類別欄位為空

**影響記錄範例**:
- 行號4: `4,金門縣烏坵鄉,小坵村,...,20,,莊金輝,...`
- 行號5: `5,金門縣烏坵鄉,大坵村,...,30,,林文彩,...`
- 行號11-17: 連江縣莒光鄉多個記錄災害類別為空

**修正建議**:
- 根據地理位置判斷可能的災害類型
- 海邊地區補填"水災、海嘯"
- 山區補填"水災、土石流"
- 一般地區補填"水災、震災"

### 4. 電話號碼格式不統一 (中等)
**問題描述**: 電話號碼存在多種格式問題

**問題類型**:
- 科學記號格式: `4.72E+11`, `2.66E+15`
- 格式不一致: `0836-8914621`, `08368914621`, `(089)731633`
- 缺少區號或格式錯誤

**修正建議**:
```regex
# 標準化電話號碼格式
# 轉換規則:
1. 科學記號 → 標準數字
2. (089) → 089-
3. 統一使用 XXX-XXXXXXX 格式
```

### 5. 預計收容人數為零 (輕微)
**問題描述**: 多個避難所預計收容人數為0

**影響記錄範例**:
- 行號11-17: 連江縣莒光鄉多個避難所收容人數為0
- 行號19-28: 桃園市觀音區多個避難所收容人數為0

**修正建議**:
- 確認是否為正確設定（某些避難所可能確實不收容一般民眾）
- 如有錯誤，根據避難所大小設定合理收容人數

### 6. 地址欄位格式問題 (輕微)
**問題描述**: 地址欄位包含多餘資訊或格式不統一

**問題範例**:
- 行號79: `"5鄰105號No.105,Zhengxing,Jinfeng Township,Taitung County 964,TaiwanR.O.C."`
- 行號60: `546365軍` (應為完整電話號碼)

**修正建議**:
- 地址欄位只保留中文地址
- 移除英文地址和郵遞區號
- 統一地址格式

### 7. 經緯度精度問題 (輕微)
**問題描述**: 部分經緯度坐標精度不足

**問題範例**:
- 行號18: `26` (緯度只有整數)
- 行號29: `27` (緯度只有整數)

**修正建議**:
- 經緯度應至少保留小數點後3位
- 使用地理編碼服務補充精確坐標

### 8. 特殊字符問題 (輕微)
**問題描述**: 欄位中包含特殊字符影響處理

**問題範例**:
- 行號79: 地址欄位包含引號和逗號
- 部分記錄包含全形括號 `（）`

**修正建議**:
- 統一使用半形符號
- 正確處理CSV中的引號轉義
