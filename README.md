# ARIA - Automated Regional Impact Auditor
# 河川洪災避難所風險評估

## 專案概述

ARIA是一個結合水利署河川圖資與消防署避難收容所資料的地理資訊分析系統，用於評估台灣各行政區的避難所洪災風險與收容量缺口。

## 分析特色

- **多級風險評估**: 建立高風險(500m)、中風險(1km)、低風險(2km)三級河川警戒緩衝區
- **真實資料分析**: 使用政府開放平台的完整資料集，非課堂Lab的少量測試資料
- **收容量缺口分析**: 不只看「在不在風險區」，更關注「收容量夠不夠」
- **互動式視覺化**: 提供動態地圖與靜態統計圖雙重展示

## 資料來源

1. **水利署河川面資料**: [水利空間資訊服務平台](https://gic.wra.gov.tw/)
2. **消防署避難收容所**: [政府資料開放平台](https://data.gov.tw/dataset/73242)
3. **鄉鎮市區界**: [國土測繪中心](https://www.tgos.tw/)

## 主要功能

### 1. 資料載入與清理
- 自動下載並清理避難所座標資料
- CRS轉換確保空間分析精度
- 移除無效座標記錄

### 2. 多級緩衝區分析
- 500m高風險緩衝區
- 1km中風險緩衝區  
- 2km低風險緩衝區

### 3. 空間連接與風險分級
- 使用gpd.sjoin()進行空間連接
- 處理一對多問題，取最高風險等級
- 風險等級: 高 > 中 > 低 > 安全

### 4. 收容量缺口分析
- 按鄉鎮市區彙總統計
- 識別安全避難所收容量不足的行政區
- 產出風險最高的Top 10行政區排名

### 5. 視覺化輸出
- 互動式風險地圖(folium)
- 靜態統計圖(matplotlib)
- 詳細的風險清單JSON

## AI 診斷日誌

### 遇到的問題與解決方案

#### 1. 資料編碼問題
**問題**: 下載的避難所CSV檔案編碼不明，導致pandas讀取失敗
**解決**: 嘗試多種編碼(UTF-8, Big5)，建立錯誤處理機制

#### 2. 座標清理挑戰
**問題**: 原始資料包含座標為0、空值、或超出台灣範圍的記錄
**解決**: 建立嚴格的座標篩選條件(經度119-122, 緯度21-26)

#### 3. CRS轉換重要性
**問題**: 課堂Lab教訓提醒必須在EPSG:3826下進行buffer分析
**解決**: 確保所有空間資料統一使用EPSG:3826，避免距離計算錯誤

#### 4. 一對多空間連接
**問題**: 避難所可能同時落在多個緩衝區內
**解決**: 實作優先級邏輯，取最高風險等級(高>中>低>安全)

#### 5. 欄位名稱變異
**問題**: 不同資料來源的欄位名稱可能不同
**解決**: 建立動態欄位識別機制，提高程式健壯性

#### 6. 視覺化挑戰
**問題**: 中文字體顯示與地圖性能優化
**解決**: 設定合適的字體備選清單，優化地圖渲染參數

### 學習心得

1. **真實資料的複雜性**: 政府開放資料雖然豐富，但需要大量的清理工作
2. **空間分析的精確性**: CRS轉換和buffer距離單位是關鍵
3. **一對多關係的處理**: 現實世界的空間關係往往比理論複雜
4. **視覺化的重要性**: 好的視覺化能讓決策者快速理解複雜的分析結果

## 使用方法

1. 確保安裝必要套件:
```bash
pip install geopandas folium mapclassify python-dotenv
```

2. 執行ARIA.ipynb筆記本，依序執行所有cell

3. 查看輸出檔案:
   - `shelter_risk_audit.json`: 詳細的避難所風險清單
   - `interactive_risk_map.html`: 可互動的地圖
   - `risk_map.png`: 靜態統計圖
   - `township_risk_analysis.csv`: 鄉鎮風險統計表

## 技術規格

- **語言**: Python 3.8+
- **主要套件**: geopandas, folium, pandas, matplotlib
- **座標系統**: EPSG:3826 (TWD97)
- **資料更新**: 2024年3月
