# ARIA - Automated Regional Impact Auditor
# 河川洪災避難所風險評估

## 專案概述

ARIA是一個結合水利署河川圖資與消防署避難收容所資料的地理資訊分析系統，用於評估台灣各行政區的避難所洪災風險與收容量缺口。

## 分析特色

- **多級風險評估**: 建立高風險(500m)、中風險(1km)、低風險(2km)三級河川警戒緩衝區
- **真實資料分析**: 使用政府開放平台的完整資料集，非課堂Lab的少量測試資料
- **收容量缺口分析**: 不只看「在不在風險區」，更關注「收容量夠不夠」
- **互動式視覺化**: 提供動態地圖與靜態統計圖雙重展示

## 資料來源

1. **水利署河川面資料**: [水利空間資訊服務平台](https://gic.wra.gov.tw/)
2. **消防署避難收容所**: [政府資料開放平台](https://data.gov.tw/dataset/73242)
3. **鄉鎮市區界**: [國土測繪中心](https://www.tgos.tw/)

## 主要功能

### 1. 環境設定與資料載入
- 使用 `.env` 檔案管理緩衝距離等參數
- 載入水利署河川面資料 (EPSG:3826)
- 載入國土測繪中心行政區界資料
- 載入消防署避難收容所資料並進行座標清理

### 2. 多級緩衝區分析
- 從 `.env` 讀取緩衝距離參數
- 500m高風險緩衝區 (BUFFER_HIGH)
- 1km中風險緩衝區 (BUFFER_MED)  
- 2km低風險緩衝區 (BUFFER_LOW)

### 3. 空間連接與風險分級
- 使用gpd.sjoin()進行空間連接
- 處理一對多問題，取最高風險等級
- 風險等級: 高 > 中 > 低 > 安全

### 4. 收容量缺口分析
- 按縣市級彙總統計 (因資料限制)
- 識別安全避難所收容量不足的行政區
- 產出風險最高的Top 10行政區排名

### 5. 視覺化輸出
- 靜態統計圖 (matplotlib) - Top 10高風險行政區
- 互動式風險地圖 (folium) - 本地生成
- 詳細的風險清單JSON
- 完整分析報告 (Markdown)

### 6. Captain's Log 文件化
- 每個分析階段前的詳細說明
- 任務目標、技術要點、預期結果
- 提升程式碼可讀性與維護性

## AI 診斷日誌

### 實作過程中的問題與解決方案

#### 1. 環境變數管理
**問題**: 緩衝距離等參數硬編碼在程式碼中，缺乏彈性
**解決**: 建立`.env`檔案統一管理環境變數，使用`python-dotenv`載入
- `BUFFER_HIGH=500` (高風險500m)
- `BUFFER_MED=1000` (中風險1km)  
- `BUFFER_LOW=2000` (低風險2km)

#### 2. 資料編碼問題
**問題**: 消防署避難所CSV檔案編碼不明，pandas讀取失敗
**解決**: 實作雙重編碼嘗試機制，優先UTF-8，失敗時自動切換至Big5

#### 3. 座標資料清理
**問題**: 原始資料包含無效座標(0,0)、空值、或超出台灣範圍的記錄
**解決**: 建立嚴格的座標篩選條件
- 經度範圍: 119-122
- 緯度範圍: 21-26
- 清理前: 5973筆 → 清理後: 5888筆 (移除85筆無效記錄)

#### 4. CRS坐標系統轉換
**問題**: 課堂Lab教訓提醒必須在EPSG:3826下進行buffer分析
**解決**: 確保所有空間資料統一使用EPSG:3826 (TWD97)，避免距離計算錯誤

#### 5. 空間連接的複雜性
**問題**: 避難所可能同時落在多個緩衝區內，產生一對多關係
**解決**: 實作風險等級優先級邏輯 (高>中>低>安全)，確保每個避難所只有一個風險等級

#### 6. 行政區資料層級問題
**問題**: 載入的行政區界資料只有縣市級資訊，無鄉鎮級TOWNNAME欄位
**解決**: 調整分析粒度至縣市級，建立佔位TOWNNAME欄位保持相容性

#### 7. 視覺化技術挑戰
**問題**: 中文字體顯示異常、地圖性能優化、互動式功能實作
**解決**: 
- 下載台北黑體字體解決中文顯示
- 設定台灣地圖邊界限制(經度119.5-122.5, 緯度21.5-25.5)
- 優化folium地圖渲染參數

#### 8. Git工作流程整合
**問題**: 需要建立專業的版本控制環境
**解決**: 建立本地Git倉庫，設定`.gitignore`排除敏感檔案(.env等)

### 技術學習心得

1. **真實資料的複雜性**: 政府開放資料雖然豐富，但需要大量清理工作
2. **空間分析的精確性**: CRS轉換和buffer距離單位是關鍵
3. **一對多關係的處理**: 現實世界的空間關係往往比理論複雜
4. **環境變數的重要性**: 彈性化的參數管理提高程式可維護性
5. **視覺化的決策價值**: 好的視覺化能讓決策者快速理解複雜分析結果
6. **版本控制規範**: 專業的Git工作流程確保協作效率

## 使用方法

1. 確保安裝必要套件:
```bash
pip install geopandas folium mapclassify python-dotenv matplotlib
```

2. 設定環境變數 (編輯 `.env` 檔案):
```bash
BUFFER_HIGH=500    # 高風險緩衝距離 (公尺)
BUFFER_MED=1000    # 中風險緩衝距離 (公尺)
BUFFER_LOW=2000    # 低風險緩衝距離 (公尺)
TARGET_COUNTY=     # 目標縣市 (空白則分析全台灣)
```

3. 執行 `run_analysis_real_data.ipynb` 筆記本，依序執行所有 cell

4. 查看輸出檔案於 `outputs/` 資料夾:
   - `shelter_risk_audit.json`: 詳細的避難所風險清單
   - `risk_map.png`: Top 10 高風險行政區統計圖
